1. INTRODUCTION
----------------------------------------------------------------

This project is a complete DevOps-integrated full-stack web
application developed using Vanilla HTML/CSS/JavaScript, Java
Spring Boot, MySQL, Redis, Docker, and Docker Compose. The main
objective of this project is to demonstrate modern DevOps
practices along with full-stack application development,
containerization, and automated CI/CD pipeline integration.

The application is designed as a Smart Task Manager system
where users can register, log in, and manage their personal
tasks with priorities, statuses, and due dates through a
clean and responsive web interface. The system uses JWT-based
authentication to ensure secure, stateless API communication.

The project integrates frontend, backend, database, and cache
services using Docker containers and manages them through
Docker Compose with health checks and inter-service networking.
It also implements complete CI/CD pipelines using both Jenkins
and GitHub Actions to automate the build, test, and deployment
lifecycle.

The project demonstrates practical implementation of DevOps
concepts such as containerization, multi-stage Docker builds,
image creation, Docker networking, named volumes, Maven build
automation, declarative Jenkins pipelines, and GitHub Actions
workflows.

4. SYSTEM ARCHITECTURE
----------------------------------------------------------------

The system follows a four-tier microservices architecture.
All tiers are deployed as independent Docker containers
connected through a shared Docker bridge network (stm-network).

  1. Frontend Layer (stm-frontend)
       - Nginx serves static HTML, CSS, and JavaScript files.
       - Acts as a reverse proxy, forwarding all /api/* requests
         to the backend container using Docker's internal DNS.
       - Exposed on host port 80.

  2. Backend Layer (stm-backend)
       - Spring Boot REST API handles business logic,
         authentication, and data access.
       - Connects to MySQL for persistent storage and Redis
         for caching.
       - Exposes /actuator/health for Docker health checks.
       - Exposed on host port 8080.

  3. Database Layer (stm-db)
       - MySQL 8 container stores user and task data.
       - Schema is auto-initialized via init.sql on first boot.
       - Data persisted using a named Docker volume (stm-db-data).
       - Not exposed to the host; only accessible internally.

  4. Cache Layer (stm-redis)
       - Redis 7 (Alpine) used for JWT session caching.
       - Data persisted with AOF (Append Only File) via a named
         Docker volume (stm-redis-data).
       - Not exposed to the host; only accessible internally.

Startup Order (enforced by Docker Compose health checks):
  stm-db (healthy) --> stm-redis (healthy) --> stm-backend --> stm-frontend

7. DEVOPS CONCEPTS IMPLEMENTED
----------------------------------------------------------------

  Docker
    - Multi-stage Dockerfile for backend (build stage: Maven
      with JDK 17; runtime stage: slim JRE Alpine image)
    - Single-stage Dockerfile for frontend (Nginx Alpine)
    - Non-root user inside containers (security best practice)
    - Named volumes for data persistence
    - Bridge network (stm-network) for inter-service communication
    - Health checks on all four containers
    - Container dependency ordering with condition: service_healthy

  Maven Build Lifecycle (CO4)
    - mvn compile  : Compiles all 12 Java source files
    - mvn test     : Runs JUnit tests using H2 in-memory DB
    - mvn package  : Packages the executable fat JAR
    - Plugins used : spring-boot-maven-plugin,
                     maven-compiler-plugin (Java 17),
                     maven-surefire-plugin

  Jenkins CI/CD Pipeline (7 Stages)
    Stage 1 : Checkout (clone from GitHub)
    Stage 2 : Build    (mvn compile)
    Stage 3 : Test     (mvn test + JUnit report)
    Stage 4 : Package  (mvn package -DskipTests)
    Stage 5 : Docker Build (build backend and frontend images)
    Stage 6 : Registry Push (push to Docker Hub)
    Stage 7 : Deploy   (docker-compose up on target server)
    Post    : Success/Failure notifications

  GitHub Actions CI/CD Workflow (3 Jobs)
    Job 1 : Build & Test  — Maven compile, test, package;
                            uploads JAR as artifact
    Job 2 : Docker Build  — Builds and pushes images to GHCR
                            (GitHub Container Registry)
    Job 3 : Deploy Sim    — Starts all services via
                            docker-compose and runs smoke test
                            (only on push to main branch)

  Git & GitHub
    - Feature branch workflow
    - Meaningful commit messages
    - .gitignore to exclude build artifacts and secrets
    - .gitattributes for cross-platform line ending normalization


8. REST API ENDPOINTS
----------------------------------------------------------------

  Authentication (Public)
    POST /api/auth/register   — Register a new user account
    POST /api/auth/login      — Login and receive a JWT token

  Tasks (Protected — requires Bearer token)
    GET    /api/tasks           — List all tasks (optional filter)
    GET    /api/tasks?status=TODO | IN_PROGRESS | DONE
    POST   /api/tasks           — Create a new task
    PUT    /api/tasks/{id}      — Update an existing task
    DELETE /api/tasks/{id}      — Delete a task
    GET    /api/tasks/stats     — Dashboard statistics

  Health (Public)
    GET /actuator/health        — Returns {"status":"UP"}
